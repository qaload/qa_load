Подключил Tinkoff gatling-kafka-plugin для отправки сообщений в Kafka (асинхронно, не request-reply).

Payload сообщения формируется динамически, поэтому в методе kafka().send() используются #{плейсхолдеры} на атрибут сессии, в котором хранится сам объект, представляющий сообщение.

То есть отправка выглядит следующим образом (Kotlin code):
exec(kafka("Kafka action").send("#{key}", "#{value}")
)

Сам библиотечный метод send() имеет сигнатуру (Java code):
public <K, V> RequestBuilder<?, ?> send(K key, V payload)

Если я все правильно понял, из-за того, что в качестве аргумента payload передается тип String (плейсхолдер на атрибут сессии), Gatling сначала достает объект из сессии по данному плейсхолдеру, а затем делает на нем вызов toString() и использует полученную строку в качестве payload. Соответственно, в топик попадает невалидная строка (в моем случае - строка из автоматически сгенерированного Kotlin toString() метода). По ходу выполнения кода создается ProducerRecord<String, String> (и ключ, и значение - строки), который далее передается в сериализатор. Сериализатор кастомный, должен конвертировать payload в массив байтов AVRO формата, но в метод serialize() приходит payload в виде той самой строки, а не сам объект, из-за чего не получается сериализовать данную строку по AVRO-формату.

Кто-нибудь знает, как можно обойти данную проблему без костылей - силами самой библиотеки?
Возможно, не увидел корректный метод, но, возможно, библиотека так заточена на работу с использованием Gatling Expression Language и любой payload будет приводить к строке.

Использование именно этой либы - необязательно, буду рад, если посоветуете библиотеку, в которой извлечение объекта из сессии работает корректно, то есть в сериализатор объект дойдет без каких-либо преобразований.
