Всем привет! Открытие вчерашнего вечера - профили в maven (http://www.apache-maven.ru/profiles.html)

Начал их использовать для проектов #JMeter. Чтобы переключаться между параметрами нагрузки. Например, длительностью - 1 час по умолчанию и 1 минута для отладки

1. Вынести параметры нагрузки в катушках и таймерах в переменные, разместить переменные в элементе Test Plan / User Defined Variables принимая значения из Properties:

Duration = ${__P(Duration,3600)}

2. Используя jmeter-maven-plugin задать значения параметров нагрузки в pom.xml тоже через properties:

<properties>
<test.Duration>3600</test.Duration>
</properties>

<build>
<plugins>
<plugin>
<groupId>com.lazerycode.jmeter</groupId>
<artifactId>jmeter-maven-plugin</artifactId>
<version>2.8.6</version>
<configuration>
<propertiesUser>
<Duration>${test.Duration}</Duration>
</propertiesUser>
...
3. Используя профили maven - переопределить некоторые properties

<profiles>
<profile>
<id>Debug</id>
<properties>
<test.Duration>60</test.Duration>
</properties>
</profile>

Теперь
тест на 1 час:
mvn jmeter:jmeter

тест на 1 минуту:
mvn jmeter:jmeter -P Debug

«только одну»

Вот вам ещё парочка:
1) Воспроизводимость результатов. У Precise повторный прогон тестов подаст ту же нагрузку. В случае «10 в час» это вообще важно.

2) Если указать «60 в час», то Constant будет подавать ровно один раз в минуту. Если тредргуппы 2, то они будут синхронно фигачить. Условно говоря, «ровно в 1:00, 2:00, 3:00 и так далее будут фигачить запросы»
В случае Precise же такой «ровной синхронизации» не будет.

3) Более того, Precise даже на небольших нагрузках (~60 в час) всё равно будет моделировать ситуации «два запроса почти подряд». Иными словами, оно может 2-3-4-5 минут подряд не посылать запросы вообще, а потом жахнуть несколько с небольшими интервалами.
Это позволяет находить concurrency баги (когда 2-3 запроса одновременно пошли).

4) Precise моделирует пуассоновский поток. Вот здесь можно почитать почему для нагрузочного тестирования пуассоновский (precise) в 100500 раз круче равномерного (constant): http://perfdynamics.blogspot.com/2012/05/load-testing-with-uniform-vs.html
