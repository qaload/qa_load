Открытой системой (в рамках тестирования производительности) - называется такая система, в которой невозможно контролировать число конкурентных пользователей. При увеличении времени отклика, пользователи, которые уже находятся в системе, продолжают выполнение своих задач, пока новые пользователи продолжают прибывать. Пример - любой онлайн-магазин, при покупках товара пользователь не ждёт другого пользователя, чтобы совершить весь свой путь по сайту. Он покупает (читай создаёт запросы) не смотря на остальных.

Закрытая система имеет обратную связь, и прибывающие пользователи становятся в очередь, если система не успевает их обслужить. Из очереди пользователь может выйти и быть поданным на вход в систему только тогда, когда существующий пользователь закончит свой "сценарий" и выйдет. При закрытой модели необходимо использовать закрытую модель рабочей нагрузки, в которой контролируется количество конкурентных пользователей. Показатель прибывающих новых пользователей не контролируется.

Использую Offset и он вычисляется при клике по ячейке таблицы

Prometheus единственный наверно движок, который позволяет добавлять вычисления с переменными Grafana за счет вызова функции label_replace и добавления переменной в результат выборки

связка такая - при старте теста в Jenkins в victoria metrics записывается момент старта теста, при завершении теста записывается момент старта и момент завершения теста

в grafana выводится две таблицы, там идентификатор теста в каждой, название стенда, версия системы, длительность

в первой таблице в скрытой колонке момент старта теста и момент завершения — при клике по любой строке значения from и to для grafana задаются из значения старта и завершения теста (переходим на интервал первого теста)

во второй таблице с помощью функции

https://prometheus.io/docs/prometheus/latest/querying/functions/#label_replace

к результатам добавляется переменная from — таким образом в таблице есть from (старт первого теста) и также есть значения старта второго теста

с помощью вкладки Transformation в Grafana вычисляется разница между from и моментом старта второго теста и хранится в скрытой колонке offset

——

При клике по любой строке во второй таблице заполняется переменная offset (переменная Grafana) делается это примерно вот так
&var-offset=${__data.fields.offset} — заполняем переменную через URL подставляя в нее значение из ячейки таблицы

/d/${__dashboard}/?${source:queryparam}&${suite:queryparam}&${test1_time:queryparam}&var-test2_time=${__data.fields.startUnix}&var-offset=${__data.fields.offset}&${__url_time_range}

offset используется чтобы сдвинуть метрики и получить интервал второго теста
