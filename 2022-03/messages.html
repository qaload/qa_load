Тут проблема в том, что, если regexp не найдётся, то «ленивость» не поможет.

При написании regexp’ов стоит обращать внимание на «повторение слишком общих» символов и заменять их на более узкий вариант.
В конкретном случае так: <option[^>]*>([^<]+)</option>
Будет работать правильнее и быстрее.

В Java (и некоторых других движках) есть ещё чуть более оптимальный режим — Possessive Quantifiers — <option[^>]*+>([^<]++)</option>, но уже не так критично (хотя есть случаи, когда оно позволяет избавиться от stackoverflow и т.п.).

Не ошибаетесь, но я бы уточнил: само по себе регулярное выражение ничего не извлекает.
Например, в JavaScript, RegExp.match всегда возвращает только одно вхождение.
Чтобы вернуть несколько нужно указывать флаг g и вызывать .exec несколько раз.

С другой стороны, String.matchAll (в том же JS) возвращает итератор всех результатов.

Конечно, в исходном варианте <option.*> действительно началось бы на первом option и скушало всё до последнего, т.е. вообще было бы невозможно найти несколько вхождений (вне зависимости от того, каким API ищем). Bogdan, you asked about tag "run", Gatling, Grafana, and InfluxDB.

I used InfluxDB 1.8 (old version)
with example config:
https://github.com/polarnik/gatling-grafana-dashboard/blob/master/devenv/influxdb.conf#L429-L436
  "v2.gatling.*.*.*.users.*.*      .measurement.run.host.simulation.measurement.request.field",
  "v2.gatling.*.*.*.*.*.*   .measurement.run.host.simulation.request.status.field   group=0",
In this case:
- "v2" prefix is skipping
- "gatling" is measurement
- first "*" is "run" tag
- second "*" is "host" tag

If you are using InfluxDB 2.0 then you could configurate:
telegraf: https://www.influxdata.com/time-series-platform/telegraf/
with config section "inputs.socket_listener":
- https://docs.influxdata.com/telegraf/v1.21/plugins/#socket_listener
- https://github.com/influxdata/telegraf/blob/release-1.21/plugins/inputs/socket_listener/README.md
with data format "graphite":
- https://github.com/influxdata/telegraf/tree/master/plugins/parsers/graphite
[[inputs.socket_listener]]
  service_address = "tcp://:2003"
  data_format = "graphite"
  separator = "."
  templates = [
  "gatling.*.users.*.*      measurement.simulation.measurement.request.field",
  "gatling.*.*.*.*   measurement.simulation.request.status.field   group=0",
  "v2.gatling.*.*.*.users.*.*      .measurement.run.host.simulation.measurement.request.field",
  "v2.gatling.*.*.*.*.*.*   .measurement.run.host.simulation.request.status.field   group=0"
  ]

- https://docs.influxdata.com/telegraf/v1.21/plugins/#influxdb_v2
- https://github.com/influxdata/telegraf/blob/release-1.21/plugins/outputs/influxdb_v2/README.md

- "gatling" is default value of rootPathPrefix
- If environment variable or property "GATLING_PREFIX" is not empty then value of "GATLING_PREFIX" is value of rootPathPrefix
- If environment variable or property "gatling.rootPathPrefix" is not empty then value of variable is value of rootPathPrefix
