}
}
Полный семпл здесь: https://github.com/savvagen/gatling-java-kotlin-demo/blob/master/src/gatling/scala/com/example/cases/ErrorHandling.scala


Для старой версии чуть все сложнее, тебе нужно будет напсать свой FailureScenario который будет наблюдать извне за упавшими реквестами или ошибками и если какойто реквест KO то он через какую-то паузу делает System exit with status code: 1

1. Сценарий:


object FailOverScenario {

var exitFlag = new AtomicBoolean(false)

val failAction: ChainBuilder =
doIf(session => exitFlag.get()) {
pause(10 seconds)
.exec(s => {
println("Found failed HTTP requests.\nExiting!")
System.exit(1)
s
})
}
}


2. Как использвать прикрепить проверку к реквесту

def exitSimulationOnFailure: ChainBuilder = {
exec((session: io.gatling.core.session.Session) => {
if(session.status == KO) {
FailOverScenario.exitFlag.set(true)
}
session
})
}

def exitIfFailed: ChainBuilder = exec(exitSimulationOnFailure).exitHereIfFailed



Сам сценарий:

exec(createComment())
.exec(exitIfFailed)
.pause(1.seconds)


как подключить сценарий к симуляции:

failOverScenario.inject(constantUsersPerSec(1).during(maxDuration.seconds))
